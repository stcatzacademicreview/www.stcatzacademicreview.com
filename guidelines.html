You can link to these tabs (but this isn't implemented using CSS :target)

Just throw in a hash to the URL and the appropriate tab is opened.

On Firefox middle clicking a tab works as one would expect. WebKits and IE register the middle click on the label and not to link.

The last visited tab is remembered in browser history. Changing tabs does not add new entries to browser history.

#one

You can link inside the tabs

The correct tab becomes visible when you use a hash that points inside a tab panel.

#two

These tabs are usable without JavaScript

Turn JS off. Still works. Minus the hash tricks. Minus old browsers like IE8.

#three

So how is this any different from the rest?

I think I haven't seen many implementations that attempt to fix usability issues related to linking inside tabs. This solution isn't perfect either, but "so simple it works".

I don't know how friendly these tabs are to people who can't see them, but hear them. The tabs however are not hidden with display: none;, technically the content is visible all the time. It only gets hidden because the container has overflow: hidden; and all the panels are 100% wide and in a row.

This solution was made to replace jQuery UI tabs that were created with Zend. Awful code to work with.
